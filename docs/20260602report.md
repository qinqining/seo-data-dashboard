**读者**：运营经理

## 一、名词：Top10 / Top30 / Top100

| 术语 | 含义 | 出现在哪里 |
|------|------|------------|
| **Top100** | 在某个国家 Google 上，该词自然排名 **第 1～100 名**（约前 10 页能搜到） | Ahrefs「有机关键词」接口本身只返回 Top100 内的词；关键词表「当前排名」为空 + 排名变化显示「未进Top100」= 该词不在 Top100 |
| **Top1–3 / 4–10 / 11–20 / 21–100** | 按 `best_position` 分档的词数 | 看板四档词数 + 各档周环比（锚点日 vs 锚点−7） |
| ~~Top10 / Top30~~ | 已由四档替代 | — |

**注意**：看板分档统计 **Ahrefs 有机词**（默认 US，最多 1000 条），与关键词表是否手工填词无关。

## 二、`run_sync` 日期与窗口（锚点 -3 等）

### 2.1 概念一：锚点日（≠ 你点 `run_sync.bat` 当天）

| 项目 | 说明 |
|------|------|
| **公式** | 锚点日 = **运行 sync 当天 − `DATA_DELAY_DAYS`**（默认 **3**） |
| **例子** | 6 月 2 日跑 → 锚点日 = **5 月 30 日**（不是 6 月 2 日） |
| **看板 7 行** | 锚点 −6 … 锚点，例如 **5/24～5/30**（每行 = 那一天的 GSC 日报） |
| **只写在锚点日一行** | Top 分档、四档周环比、RD、收录汇总、**周环比流量** 等 |

关键词 / 页面 / 外链表的 **数据日期** 一般也是锚点日，不是运行当天。

### 2.2 概念二：周环比（两种算法，都写在锚点日那一行）

**不是**日历「本周一～日 vs 上周一～日」，而是下面两种。

#### A. 周环比流量（GSC，两段各 7 天）

锚点 = **5/30** 时：

| | 日期范围 | 含义 |
|--|----------|------|
| **本期** | 5/24～5/30 | 最近 7 天全站点击 **合计** |
| **上期** | 5/17～5/23 | 再往前 7 天点击 **合计** |

写入：` (本期 − 上期) ÷ 上期 `（小数，如 0.2=涨 20%，−1=跌 100%）。**上期点击 = 0 则不写。**

#### B. 周环比 Top1-3 / 4-10 / 11-20 / 21-100（Ahrefs，锚点日 vs 锚点−7 天）

一次 API 带两个日期：

| API 参数 | 锚点 5/30 时 | 用来干什么 |
|----------|--------------|------------|
| `date` | 5/30 | 每个词的 `best_position` → 数 **本期** 各档词数 |
| `date_compared` | **5/23**（锚点 **减 7 天**） | 每个词的 `best_position_prev` → 数 **上期** 各档词数 |

各档周环比 = `(5/30 该档词数 − 5/23 该档词数) ÷ 5/23 该档词数`。**5/23 该档词数 = 0 则不写**（所以 Top4-10 周环比可能为空）。

> 易混：对比日是 **5 月 23 日**，不是「减 7 个月」；也不是单日 5/30 对比单日 5/23 的**全站点击**（那是流量周环比的事）。

### 2.3 看板 7 天回填（对照表）

| 配置项 | 默认 | 含义 |
|--------|------|------|
| `DASHBOARD_SYNC_DAYS` | **7** | 写入锚点 −6 … 锚点，每站每天一行 |

| 看板行 | 写入内容 |
|--------|----------|
| 前 6 天（如 5/24～5/29） | 仅 GSC：点击、展示、CTR、平均排名 |
| **锚点日**（如 5/30） | GSC 四项 + 四档词数 + 四档周环比 + 周环比流量 + RD + 收录 + 异常预警 |

### 2.4 其他指标日期

| 指标 | 区间 |
|------|------|
| 近 7 天 RD 变化 | Ahrefs：锚点 −7 ～ 锚点，referring domains **期末 − 期初** |

### 2.5 其他常用 `.env`（与日期/数据范围相关）

| 变量 | 默认 | 作用 |
|------|------|------|
| `AHREFS_TARGET_COUNTRY` | `us` | 看板分档 / 关键词用的 **Google 国家**（`all` = 多国合并） |
| `AHREFS_BACKLINKS_LIMIT` | `500` | 每站外链 API 最多导入条数 |
| `CACHE_ENABLED` / `CACHE_TTL_HOURS` | 开 / 12 | GSC 7 日 batch 本地缓存；近期 clicks=0 会在约 `DATA_DELAY_DAYS + GSC_RECENT_REFRESH_DAYS` 天内自动重拉 |
| `SYNC_SITES` | 空=五站全跑 | 如 `cncpioneer,fecision` 只跑指定站 |

### 2.6 分表跑命令（备忘）

```bat
REM 只跑页面管理表
run_sync.bat --skip-keywords --skip-backlinks --skip-dashboard

REM 只跑站点关键词库（示例：跳过 richconn）
run_sync.bat --skip-pages --skip-backlinks --skip-dashboard --site cncpioneer --site fecision --site lasermicrofab --site drametal

REM 只刷新看板（需先跑页面或同次未 skip-pages，收录数才准确）
run_sync.bat --skip-keywords --skip-pages --skip-backlinks
```

## 三、表 1：SEO 自动数据看板

**脚本表名**：`SEO自动数据看板`  
**Upsert 键**：`日期` + `独立站`（每站每天一行）  
**运营是否要填**：否，全部自动写入。

| 字段名（明道云） | 来源 | 含义说明 |
|------------------|------|----------|
| 日期 | 脚本 | 该行指标所属日期（非点击发生的「录入时间」） |
| 独立站 | 脚本 | 五站之一：cncpioneer / fecision / richconn / lasermicrofab / drametal |
| 自然点击 | **GSC** | 该日全站自然搜索 **点击次数** |
| 展示量 | **GSC** | 该日全站 **展示次数** |
| 平均CTR | **GSC** | 点击率（脚本写入 0～1 小数，如 0.05 ≈ 5%） |
| 全站加权平均排名 | **GSC** | 全站 **加权平均排名**，不是「核心词平均排名」 |
| Top1-3词数 | **Ahrefs** + 脚本 | 排名 1–3 |
| Top4-10词数 | **Ahrefs** + 脚本 | 排名 4–10 |
| Top11-20词数 | **Ahrefs** + 脚本 | 排名 11–20 |
| Top21-100词数 | **Ahrefs** + 脚本 | 排名 21–100（有机词 API 上限） |
| 近7天RD变化 | **Ahrefs** | 近 7 天 RD 净值变化 |
| 已监控URL收录数 | **GSC** + 脚本 | 页面表已填 URL 且已收录 |
| 已监控URL异常数 | **GSC** + 脚本 | 未收录或索引异常 |
| 异常预警 | **脚本** | 周环比流量 ≤ −20% → 流量下跌 |
| 周环比流量 | **GSC** + 脚本 | 近 7 天点击 vs 再前 7 天 |
| 周环比Top1-3词 … Top21-100词 | **Ahrefs** + 脚本 | 各档词数 vs 锚点−7 天，分母为 0 不写入 |

**易误解点**

1. 前 6 天行只有 GSC 四项，**没有** 分档词数、RD、收录数。  
2. 「收录数/异常数」依赖 **页面表是否已填 URL**；未填则长期为 0。  
3. GSC 单日点击为 0、但周合计有数，常见于数据延迟，不一定是站没流量。

## 四、表 2：站点关键词库

**脚本表名**：`站点关键词库`  
**Upsert 键**：`独立站` + `关键词`（一词一行）  
**导入方式**：Ahrefs 有机词 **自动新建/更新**；运营 **不必先手工建词**。

| 字段名（明道云） | 来源 | 含义说明 |
|------------------|------|----------|
| 独立站 | 脚本 | 写入时带站点单选 |
| 关键词 | **Ahrefs** | 搜索词文本 |
| 搜索量 | **Ahrefs** | 月搜索量 `volume` |
| KD | **Ahrefs** | 关键词难度 0–100 `keyword_difficulty` |
| 当前排名 | **Ahrefs** | 该国 Google **最佳自然排名**；仅 Top100 内才有值 |
| 排名变化 | **脚本**（依据 Ahrefs 7 日对比） | `↑3` / `↓2` / `持平` / **未进Top100** |
| 排名落地页 | **Ahrefs** | 该词当前最佳排名 URL `best_position_url` |
| 数据日期 | 脚本 | 一般为 **锚点日** |
| 优先级 / 关键词类型 / 优化状态 / 关联落地页 等 | **人工** | 分级与计划，脚本 **不覆盖** |

**API 参数要点**（供技术对照）

- 接口：`site-explorer/organic-keywords`  
- 国家：`AHREFS_TARGET_COUNTRY`（默认 `us`）  
- 单次最多 **1000** 条（`limit=1000`）  
- 返回 0 条 → 日志 `Loaded 0 organic keywords`：表示 **该国 Top100 内无词**，不是脚本过滤掉了有排名的词。

## 五、表 3：页面管理表

**脚本表名**：`页面管理表`  
**Upsert 键**：`独立站` + `页面URL`  
**导入方式**：**只更新已有行**，不自动新建 URL。

| 字段名（明道云） | 来源 | 含义说明 |
|------------------|------|----------|
| 独立站 | **人工** | 筛选本站行；脚本按此字段查行 |
| 页面URL | **人工** | **必填**；须为完整 `https://...`，与 GSC 资源一致 |
| 收录状态 | **GSC** | URL Inspection：已收录 / 未收录 / 索引异常 |
| 页面流量 | **GSC** | 锚点日该 URL 的 **自然点击**（非展示） |
| 数据日期 | 脚本 | 一般为锚点日 |
| 页面类型 / 关联关键词 / 优化记录 / 质量校验 等 | **人工** | 脚本不写 |

**运营必做**：每站至少录入要监控的首页或重点落地页 URL；否则同步报告会出现 `[SKIP] 页面表无行`，看板「已监控URL收录数」也为 0。

## 六、表 4：外链监控表

**脚本表名**：`外链监控表`  
**Upsert 键**：`独立站` + `来源URL` + `目标URL`  
**导入方式**：Ahrefs **自动新建/更新**；运营 **不必先手工填来源域名**。

| 字段名（明道云） | 来源 | 含义说明 |
|------------------|------|----------|
| 独立站 | 脚本 | 站点单选 |
| 来源域名 | **Ahrefs** + 脚本 | 从来源 URL 解析的主机名 |
| 来源URL | **Ahrefs** | 外链所在页面 `url_from` |
| 目标URL | **Ahrefs** | 指向本站的 URL `url_to` |
| 锚文本 | **Ahrefs** | `anchor` |
| 域名DR | **Ahrefs** | 来源域 DR `domain_rating_source` |
| 是否Dofollow | **Ahrefs** | 是 / 否 |
| 首次发现 | **Ahrefs** | `first_seen` 日期 |
| 外链存活 | **Ahrefs** | 有效 / 已失效（对应 live / lost，**不是** Google 是否收录该页） |
| 数据日期 | 脚本 | 一般为锚点日 |
| 行业相关性 / 审核结果 / 违规说明 等 | **人工** | 主管审核，脚本不写 |

**条数上限**：每站 `AHREFS_BACKLINKS_LIMIT`（默认 500）；API 使用 `history=live`，`aggregation=1_per_domain`（每域一条代表）。

## 七、不纳入本报告的 2 张表（纯人工）

| 表名 | 脚本是否写入 | 说明 |
|------|----------------|------|
| SEO 每日执行日志 | **否** | 运营每日动作记录 |
