### 1.2 页面管理表 — 现状与建议（讨论，未改代码）

**脚本今天能做什么（`sync_pages`）：**

| 能力 | 说明 |
| --- | --- |
| **URL 录入** | **不**从 GSC Top / Ahrefs 批量建重点页；仅 `config/sites.json` → `homepage_url` **无首页行时自动 add 一行** |
| **已有行更新** | 按 `页面URL`（+ 表内 `独立站` 筛选）逐行 **editRow** |
| **GSC 自动字段** | **收录状态**（URL Inspection）、**页面流量**（锚点日该 URL 点击）、**数据日期** |
| **看板联动** | 锚点日 **已监控URL收录数 / 异常数** = 页面表本轮 indexed / issues 计数（skip pages 则为 0） |
| **Ahrefs** | **本期未接**页面表 |

**GSC 可补、Ahrefs 可补（若二期扩展页面表）：**

| 来源 | 建议自动列 | 说明 |
| --- | --- | --- |
| GSC Search Analytics（按 URL） | 展示量、CTR、平均排名 | 与 GSC Top 页面同 API 维度，**锚点日或 7 日** |
| GSC（脚本计算） | 7 日点击合计 | 重点页看周流量，比单日点击稳 |
| GSC URL Inspection | （已有）收录状态 | 配额紧，仅 **已登记 URL** |
| Ahrefs `top-pages` / URL 指标 | 自然流量估值、UR、该 URL 有机词数 | 与 GSC 点击口径不同，作参考 |
| Ahrefs `all-backlinks` 过滤 `url_to` | 指向该 URL 的 RD 数 / 外链条数 | 页面级外链质量，非全站 DR |
| Ahrefs `organic-keywords` 聚合 | 排名词数、Top10 词数 | 可从关键词表视图关联，**不必重复进页面表** |

**表结构建议（保持与 `docs/dashboard.md` §4.3 分工）：**

- **页面管理表** = 运营登记的 **重点 URL**（页面类型、关联词、优化记录、质量校验）+ **GSC 收录/流量**（现脚本已写三列 API 字段）
- **GSC Top 页面明细** = 全站 Top N 发现层，**不替代**页面表；勿把 Top 表几千 URL 同步进页面表
- **URL 批量录入**：明道云导入 / 从 GSC Top 视图人工挑 URL 粘贴；若要做「从 Top 表 seed 重点页」，单独开发 **可选** `--seed-pages-from-gsc-top N`，默认关闭
- 明道云字段权限：API 列只读，人工列运营可写

### 单站流水线（完整）

```
GSCClient(site) + AhrefsClient(anchor)
  ├─ sync_keywords           → 站点关键词库
  ├─ sync_pages              → 页面管理表（仅更新已有行 + 可选 seed 首页）
  ├─ sync_backlinks          → 外链监控表
  ├─ sync_gsc_top_queries    → GSC Top 查询（7 日 × query×country）
  ├─ sync_gsc_top_pages      → GSC Top 页面（7 日 × page×country）
  └─ sync_dashboard          → 看板 7 日 GSC + 按日四档/本站DR；锚点日：Backlinks、收录/异常、周均值等
```
