## 1. 最新架构与核心逻辑

### 入口与配置

| 层 | 路径 | 职责 |
|----|------|------|
| 入口 | `run_sync.bat` → `script/sync.py` | 五站循环 sync，写报告 `reports/` |
| 站点 | `config/sites.json` | `key` / `gsc_site_url` / `ahrefs_domain` |
| 明道云字段 | `config/mingdao_options.json`（看板） | `config/mingdao_worksheets.json`（关键词/页面/外链） |
| 密钥 | `.env` | 明道云 API、Ahrefs Token、GSC OAuth、`DATA_DELAY_DAYS` 等 |

### 单站流水线（`sync_site`）

```
GSCClient + AhrefsClient(anchor_date)
  ├─ sync_keywords      → 站点关键词库（Ahrefs organic-keywords, upsert by 关键词）
  ├─ sync_pages         → 页面管理表（GSC 页面点击 + URL Inspection，仅更新已有行）
  ├─ sync_backlinks     → 外链监控表（Ahrefs all-backlinks, upsert by 来源URL+目标URL）
  └─ sync_dashboard     → SEO 自动数据看板（7 天 × 1 行/天/站）
```

### 看板时间窗口

- **锚点日** = `today - DATA_DELAY_DAYS`（默认 **3**，覆盖 GSC 1–3 天延迟）
- **回填** = `DASHBOARD_SYNC_DAYS`（默认 **7**）：`锚点-6 … 锚点`
- **GSC**：一次 batch `dimensions=date` 拉 7 天 → 写入 `cache/gsc/{site}.json`
- **Ahrefs / 周环比 / 页面收录汇总**：仅写在**锚点日**那一行；前 6 天只写 GSC 四指标

### 网络

- **GSC**：Clash 系统代理（`.env` 不设 `HTTPS_PROXY`）
- **Ahrefs / 明道云**：直连（可选 `MINGDAO_PROXY`）

### CLI 粒度控制

```bat
--site richconn              # 单站（可重复）
--skip-keywords|--skip-pages|--skip-backlinks|--skip-dashboard
--refresh                    # 忽略 GSC 缓存强制重拉
SYNC_SITES=cncpioneer,fecision  # .env 过滤
```

## 2. 已解决的关键 Bug 与修复

| Bug | 根因 | 修复 |
|-----|------|------|
| Ahrefs **400** `country=all` | API 不支持 `all` | `.env` 改 `AHREFS_TARGET_COUNTRY=us`；可选 `all` + 多国 merge |
| GSC **403**（richconn 等三站） | `gsc_site_url` 与 Search Console 资源不一致 | 改为 `sc-domain:xxx.com` |
| `sc-domain:` 被加 `https://` | `normalize_gsc_site_url` 误处理 | 识别 `sc-domain:` 前缀，原样保留 |
| 看板 7 天写入同一行 | 明道云日期筛选 `filterType=17` 失效 | 改为 **`filterType=2`**（日期精确匹配） |
| 看板 CTR/点击常 **0** | `DATA_DELAY_DAYS=2` 过短 + 单日查询 | 默认 **3** + 7 天 batch + `SyncCache`（近期 0 值自动重拉） |
| 页面表 **SKIP 无行** | sync 时明道云尚未录入 URL | 运营先填表；脚本支持 URL 自动补 `https://` |
| 外链/关键词「看不到数据」 | 视图筛选 / 分页；非写入失败 | API 核实五站均有行；报告 `[WRITE]` 可对账 |
