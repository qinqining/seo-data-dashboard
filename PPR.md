# 飞书多维表格 SEO 数据自动化落地方案

## 1. 背景

当前项目文件夹下已有 Word 文档：

`机加工外贸SEO运营管理（飞书多维表格完整版）.docx`

该文档设计了一套面向机加工外贸 SEO 运营的飞书多维表格体系，目标是通过 GSC API、Ahrefs API 和飞书多维表格 API，实现 SEO 数据自动同步、运营动作量化、质量审核和周/月绩效追踪。

本方案基于当前对话确认：

- GSC API 和 Ahrefs API 负责读取数据。
- 飞书多维表格 API 负责写入数据表。
- 需要一个中间轻量脚本负责定时拉取、清洗、匹配、写入。
- 第一版可部署在运营经理 Windows 电脑上，通过 `.bat` 安装和 Windows 定时任务每日触发。
- 飞书权限建议：运营经理一人编辑，其他人阅读；自动字段由飞书自建应用写入。

## 2. 总体架构

```text
运营经理 Windows 电脑
  |
  | Windows Task Scheduler 每日触发
  v
run_daily.bat
  |
  v
sync.py
  |
  +-- Google Search Console API: 点击、展示、CTR、排名、页面流量、URL 检查
  +-- Ahrefs API: 搜索量、KD、关键词排名、Top10/Top30、DR、外链数据
  |
  v
飞书多维表格 API
  |
  v
SEO 自动数据看板 / 关键词库 / 页面表 / 外链表 / 日志表 / 周复盘表
```

## 3. API 能力边界

### 3.1 GSC API

GSC API 可以读取 Google Search Console 中的网站搜索表现数据，适合用于：

- 自然点击
- 展示量
- 平均 CTR
- 平均排名
- 按关键词查询表现
- 按页面 URL 查询表现
- URL Inspection 检查指定 URL 的索引状态

注意：

- GSC 数据通常有延迟，不应按实时数据设计。
- URL Inspection API 适合按 URL 检查，不适合一次性获取全站精确收录总数。
- 建议将“网站总收录页面”改成“已监控 URL 中已收录数量”或“页面库 URL 收录状态统计”。

### 3.2 Ahrefs API

Ahrefs API 可以读取 SEO 第三方数据，适合用于：

- 关键词搜索量
- Keyword Difficulty
- 当前关键词排名
- Top10 / Top30 关键词数量
- Referring Domains
- Domain Rating
- 外链状态

注意：

- Ahrefs API 需要账号套餐支持。
- API 调用会消耗配额，需要控制同步频率。
- 第一版建议每日同步一次，不做高频刷新。

### 3.3 飞书多维表格 API

飞书多维表格 API 负责：

- 新增记录
- 更新记录
- 查询记录
- 字段映射
- 表结构维护

注意：

- GSC 和 Ahrefs 不能直接写飞书，需要通过脚本调用飞书 API。
- 飞书写入建议使用自建应用的 `app_id` 和 `app_secret`，不是使用运营经理个人账号密码。
- 自建应用需要被授权访问目标多维表格。

## 4. 权限方案

### 4.1 飞书多维表格权限

| 角色 | 权限 |
| --- | --- |
| 运营经理 | 编辑 |
| 其他运营 | 阅读或受限编辑，视实际管理规则决定 |
| 老板/管理层 | 阅读 |
| 飞书自建应用 | API 读写 |

### 4.2 字段编辑原则

- 自动数据字段由脚本写入，运营不要手动修改。
- 运营只填写执行类字段，例如页面优化记录、外链来源、每日动作、周复盘。
- 主管或运营经理负责审核字段，例如质量校验结果、审核结果、周考核等级。

## 5. 部署方案

第一版采用运营经理 Windows 电脑本地部署。

### 5.1 文件结构

```text
D:\seo-data-dashboard
  ├─ PPR.md
  ├─ .env
  ├─ .env.example
  ├─ .gitignore
  ├─ requirements.txt
  ├─ install.bat
  ├─ run_daily.bat
  ├─ sync.py
  ├─ google_credentials.json
  ├─ client_secret.json
  └─ logs\
```

### 5.2 运行方式

1. 运营经理电脑安装 Python。
2. 双击 `install.bat`：
   - 创建 Python 虚拟环境。
   - 安装依赖。
   - 创建 `logs` 文件夹。
   - 创建 Windows 每日定时任务。
3. 打开 `.env`，补充密钥和表格 ID。
4. 每天定时运行 `run_daily.bat`。
5. `sync.py` 读取 API 数据并写入飞书多维表格。

### 5.3 风险

- 运营经理电脑必须开机、联网，且不能休眠。
- 本地脚本需要妥善保护 `.env`、`google_credentials.json`、`client_secret.json`、`token.json`。
- 如果后续对稳定性要求更高，建议迁移到云服务器、公司 Windows Server、NAS、GitHub Actions 或云函数。

## 6. 多维表格需求与字段

## 6.1 表 1：SEO 自动数据看板

用途：自动同步 Ahrefs 和 GSC 数据，无人工录入，杜绝数据造假。

飞书设置：开启自动统计视图，按周/月汇总趋势，异常数据标红预警。

| 字段名称 | 字段类型 | 数据来源 | 备注/自动规则 |
| --- | --- | --- | --- |
| 日期 | 日期 | 系统自动 | 每日自动生成 |
| 自然点击（GSC） | 数字 | GSC API | 每日网站自然搜索点击量 |
| 展示量（GSC） | 数字 | GSC API | 每日搜索展示次数 |
| 平均CTR | 百分比 | GSC API | 系统自动计算：点击/展示 |
| 全站平均排名 | 数字 | GSC API | 核心关键词平均排名 |
| 核心词Top10数量 | 数字 | Ahrefs API | 机加工高转化词进谷歌前10数量 |
| 核心词Top30数量 | 数字 | Ahrefs API | 机加工高转化词进谷歌前30数量 |
| 今日新增RD（referring domain） | 数字 | Ahrefs API | 每日新增优质外链域名数 |
| 网站总收录页面 | 数字 | GSC API | 谷歌已收录页面总数 |
| 抓取异常数量 | 数字 | GSC API | 未收录/404/加载失败页面数 |
| 异常预警 | 单选 | 系统自动 | 正常/排名下跌/流量下跌/收录异常/外链异常 |
| 周环比流量 | 百分比 | 系统自动 | 本周点击 vs 上周点击环比 |
| 周环比Top30词 | 百分比 | 系统自动 | 本周 Top30 词 vs 上周净增长 |

建议自动写入字段：

- 日期
- 自然点击（GSC）
- 展示量（GSC）
- 平均CTR
- 全站平均排名
- 核心词Top10数量
- 核心词Top30数量
- 今日新增RD（referring domain）
- 网站总收录页面或已监控 URL 收录数
- 抓取异常数量
- 异常预警
- 周环比流量
- 周环比Top30词

用途：统一管理机加工行业高转化词，规范运营选词动作，杜绝无效词优化。

飞书设置：开启筛选视图，仅展示 P0/P1 级词汇，运营每日必须从该词库选取主攻词。

| 搜索量（月） | 数字 | Ahrefs API | 自动同步，优先做≥50搜索量词汇 |
| KD难度 | 数字 | Ahrefs API | 自动同步，按难度分优先级 |

- 当前排名
- 优化次数

- 优化状态

## 7. 同步逻辑设计

### 7.1 每日同步

每天定时执行：

1. 读取 `.env` 配置。
2. 获取飞书 tenant access token。
3. 拉取飞书关键词库、页面表、外链表中的人工维护数据。
4. 查询 GSC 最近一个可用日期的数据。
5. 查询 Ahrefs 关键词和外链数据。
6. 写入或更新 SEO 自动数据看板。
7. 更新关键词库自动字段。
8. 更新页面表自动字段。
9. 更新外链表自动字段。
10. 写入日志。

### 7.2 每周同步

每周日或周一执行：

1. 汇总每日执行日志。
2. 汇总外链合格 RD 数。
3. 汇总页面新增/优化数量。
4. 对比自动数据看板中的 Top30 变化。
5. 写入周绩效复盘表。

### 7.3 Upsert 规则

| 表 | 唯一键 |
| --- | --- |
| SEO 自动数据看板 | 日期 |
| 核心关键词总库 | 关键词内容 |
| 页面全生命周期管理表 | 页面URL |
| 外链建设管控表 | 外链来源域名 + 外链指向页面URL |
| SEO 每日执行日志 | 日期 + 运营人 |
| 周绩效复盘 | 周次 + 运营人 |
